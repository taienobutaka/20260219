# 納品ファイルについて

お届けしたファイルは次の 5 点です。

------------------------------------------------------------
フォルダ名：delivery/
内容：ホームページの元データ（プログラム一式）です。
　　　`npm install` → `npm run dev` で開発サーバーを起動できます。
　　　開発会社様・開発担当者様に共有してください。
------------------------------------------------------------
フォルダ名：delivery_out/
内容：完成したホームページの静的エクスポートデータです。
　　　この中に「out」フォルダがあり、その中身をサーバーにアップロードすると公開できます。
------------------------------------------------------------
フォルダ名：delivery_public/
内容：写真や動画などの大きな素材ファイルです。
　　　必要に応じて運用環境の `public/` 直下または CDN に配置してください。
------------------------------------------------------------
ファイル名：delivery_guide.txt
内容：納品ファイルの受け取り方・展開手順について記載しています。
　　　クライアント様向けの簡易説明書です。
------------------------------------------------------------
ファイル名：DELIVERY_OVERVIEW.md
内容：開発会社様・技術担当者様向けの詳細ドキュメントです。
　　　構成・動作手順・ハッシュ値などの技術情報を記載しています。
------------------------------------------------------------

【1. フォルダ構成について】
本ZIPには、上記5項目がそのまま展開された状態で含まれています。
それぞれのフォルダ・ファイルを同階層に配置したままご利用ください。

【2. 公開用サーバーへのアップロード（delivery_out/out/）】
1. 「delivery_out」フォルダを開き、「out」フォルダをさらに開きます。
2. 「out」フォルダの中にあるファイルやフォルダ（index.html など）をすべて選択し、
   サーバーの公開ディレクトリ（例：/var/www/html や public_html）にコピーしてください。
3. コピー後、ブラウザで公開URLを開き、ページが正しく表示されるかご確認ください。
4. サーバー担当者にお渡しする場合は、
   「delivery_out/out フォルダの中身をそのまま公開フォルダに置いてください」とお伝えください。

【3. 他社に動作確認を依頼する場合】
1. 展開したフォルダ一式を、サーバー管理会社・開発会社などの担当者に送付してください。
2. 伝達時の文例：
　「delivery_out/out の内容をそのまま公開用サーバーに配置すればサイトが表示されます。」
3. 技術担当者向けには README.md および DELIVERY_OVERVIEW.md をあわせて共有いただくとスムーズです。
